# Stitch Integration Workflow

## Overview
The `client/Stitch` folder contains HTML/CSS designs generated by Stitch. We need to manually port these to React components.

## Integration Steps

### 1. Identify Source
- Locate the relevant folder in `client/Stitch/`.
- Open `code.html` to view the structure.
- Open `screen.png` to verify the visual intent.

### 2. Prepare React Component
- Create the target file (e.g., `src/pages/Home.jsx`).
- Copy the `<body>` content from `code.html`.

### 3. Syntax Conversion
- **Class Attributes**: Rename `class="..."` to `className="..."`.
- **Self-closing Tags**: Ensure `<img>`, `<input>`, `<br>` are closed (user `<img />`).
- **Styles**: If `style="..."` attribute is used, convert to JS object `style={{Prop: 'value'}}` (Rare in Tailwind).
- **Comments**: Convert `<!-- -->` to `{/* */}`.

### 4. Dynamic Data
- Replace hardcoded text with props or state variables.
- Replace `src="..."` in images with dynamic URLs or imported assets.

### 5. Component Extraction
- Identify repeating patterns (e.g., specific Product Cards, Buttons).
- Extract them into `src/components/`.

### 6. Tailwind Config
- `code.html` often has a script-based Tailwind config.
- Ensure the custom colors (`primary`, `background-light`, etc.) are added to `client/tailwind.config.js`.

```javascript
// Example tailwind.config.js update based on Stitch
theme: {
  extend: {
    colors: {
        primary: "#d70018",
        "background-light": "#f4f6f8",
        "background-dark": "#0f172a",
    }
  }
}
```
